context-window tokens that Claude must read and process. This benchmark quantifies:

| Metric | Meaning |

| **Tokens** | Estimated tokens Claude reads (`chars ÷ 4`) |
| **Cost/call** | Token cost at Claude Sonnet 4.6 input pricing |
| **Context %** | % of 200K context window consumed by one response |

## Recommendations for Claude Code agents

1. **Use `api_filtered` for large data queries** — drastically reduces token burn and context pressure.
2. **Use `cli_json` over `cli_table`** — structured data is easier for Claude to parse;    table formatting adds markup tokens with no semantic value.
3. **Use `cli_table` only for quick human-readable spot checks** — highest token cost.
4. **Always pass `--limit`** when using CLI commands to bound payload size.
5. **Prefer direct API calls via SysdigClient** when writing automation —    field-selection brings context cost to near-zero for summary questions.
